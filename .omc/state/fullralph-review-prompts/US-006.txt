## Story US-006: UI IPC配線修正: 設定保存・WebSpeech再生・エラー処理・ストア最適化
src/lib/ipc.ts, src/lib/stores.svelte.ts, src/App.svelte のみを編集（Settings.svelte と CommentList/Item と types.ts は触らない）。
1) [CRITICAL] ipc.ts: setConfig が存在しないコマンド set_config を arg {config} で叩いている。Rust は update_config(new_config) = JS側 invoke('update_config', { newConfig: config })。修正: 正しいコマンド名と引数キーに直す。setConfig の公開シグネチャ(setConfig(config))は維持し、Settings.svelte が無変更で動くようにする。
2) [CRITICAL] WebSpeech未実装: Rustは app.emit('tts-speak', text) しているがUIに listen('tts-speak', ...) も speechSynthesis も無い。修正: ipc.ts に tts-speak リスナを追加し speechSynthesis.speak(new SpeechSynthesisUtterance(payload)) を呼ぶ。stores.svelte.ts / App.svelte の初期化(chatリスナ登録箇所)から起動し、unlisten クリーンアップも行う。Tauri非在(ブラウザ)時は no-op ガード。
3) [HIGH] ipc.ts: 変更系invoke(setConfig/addChannel/removeChannel/hideMessage)が reject を握りつぶす。呼び出し元がエラーを扱えるよう reject を投げ直す/返すAPIにする（Settings側のtry/catchはUS-007で対応。ここはエラーが伝播する形にすることが目的。ブラウザmodeの null 返しは維持）。
4) [MEDIUM] stores.svelte.ts: visibleMessages の $derived がバッファ全件(最大2000)を毎回再フィルタし fragments.map().join('') を都度確保。修正: push時に各メッセージの検索用小文字文字列を一度だけ前計算してキャッシュし、フィルタ/検索が無い時は早期に _buf(− hidden)を返す。
5) [MEDIUM] stores.svelte.ts: hiddenIds が buffer eviction 後も枝刈りされず無制限に増える。eviction時に _buf に存在しないidをhiddenIdsから除去する。
6) [LOW] stores.svelte.ts: 未使用のモジュールトップレベル re-export(visibleMessages/totalCount)を削除。
Svelte5 runes($state/$derived/$effect)を正しく使うこと。ipc.ts/stores.svelte.ts/App.svelte 以外を変更しないこと。

## Acceptance criteria (verify EACH)
  1. setConfig が invoke('update_config', { newConfig: config }) を呼び、公開シグネチャ setConfig(config) は不変
  2. tts-speak リスナが登録され WebSpeech選択時に speechSynthesis.speak が呼ばれる、unlistenクリーンアップとTauri非在ガードあり
  3. 変更系invokeのrejectが呼び出し元に伝播する（握り潰さない）、ブラウザmodeのnull返しは維持
  4. 検索用文字列がpush時に前計算され、無フィルタ時は早期returnで全件再走査しない
  5. hiddenIds が eviction時に枝刈りされ無制限増加しない
  6. ipc.ts/stores.svelte.ts/App.svelte 以外を変更していない
