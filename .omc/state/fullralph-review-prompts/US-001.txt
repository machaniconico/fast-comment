## Story US-001: Twitch source 堅牢化 (received_data安定判定 / emote index / NICK一意化)
src-tauri/src/sources/twitch.rs のみを編集。3点修正:
1) [HIGH] received_data が CAP ACK / welcome numeric (001/002/CAP) など制御Textフレームでも true 化し、指数バックオフの安定判定が『接続できたら常に安定』に劣化している。修正: PRIVMSG を正規化して tx.send が成功した時だけ received_data=true にする。handle_line の戻り値を拡張して『PRIVMSGを実際にemitしたか』を呼び出し側へ伝え、それを基にバックオフをリセットすること。制御行ではリセットしない。
2) [MEDIUM] emotes タグのインデックスは Twitch では UTF-16 コードユニット基準（サロゲートペアを含む本文でズレる）。現状 chars()(コードポイント)で切っているため絵文字混在本文でエモート境界がずれる。修正: 本文を encode_utf16 でインデックスし start..=end を UTF-16 単位で切り出して String に戻す。誤っている前提コメントも修正。
3) [LOW] justinfan NICK が subsec_nanos 由来で複数チャンネル同時起動時に衝突し得る。AtomicUsize 静的カウンタを混ぜて一意化（crate追加不要）。
借用・所有権・呼び出しサイト整合を厳密に保つこと。WSLにcargo無しのためコンパイルは静的に正しさを担保。

## Acceptance criteria (verify EACH)
  1. received_data (または相当の安定フラグ) は PRIVMSG が正規化されemit成功した時のみ true になり、CAP ACK や welcome numeric では立たない
  2. emote 切り出しが encode_utf16 ベースのインデックスで行われ、サロゲートペアを含む本文でも境界がずれない
  3. justinfan NICK 生成に AtomicUsize 等の単調カウンタが混ざり同時起動でも衝突しない
  4. twitch.rs 以外のファイルを変更していない、PING→PONG 応答ロジックは維持されている
